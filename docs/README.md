## Regola operativa

Ogni conversazione che cambia una di queste cose deve aggiornare almeno un documento:

- obiettivo del prodotto
- scelta architetturale
- stato operativo
- roadmap o next steps
- decisione tecnica rilevante

Se non cambia nulla di sostanziale, non serve aggiornare la doc.
